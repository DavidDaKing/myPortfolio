Vulnerability Analysis

Identified Vulnerabilities

  • Reverse Tabnabbing: A web application vulnerability that allows an attacker to replace a legitimate page with a phishing webpage. This leads to credential harvesting or sensitive information being leaked because the user may not notice that the site has been changed to a phishing site.

Exploitation Vector

Links within this website use target="_blank" without the implementation of rel="noopener" which shows that this is a viable attack vector.

Post-Exploitation

Lateral Movement from Danny --> Adrian

When enumerating the initial account I have access to, I found that Daniel is apart of a custom group called Administrators. I issued the find command to search for the files that I have access to under the administrators group. In doing this, I found a python script under the home directory of a user named Adrian. The python code is telling me that it is executing a repeating task which checks the webpage availability. Further examining the home directory of Adrian validates my suspicion because it shows a text file of every time the file executes.

Proof of Compromise

Since I have full modification access to the python file, I can add code which sends me a shell into Adrian's account

Privilege Escalation from Adrian --> Root

Enumerating sudo permissions led me to find a vulnerability where Adrian is able to get into a root shell due to being granted sudo permissions for the vim binary.

Proof of Compromise

I am able to exploit this vulnerability through a one-line command which is 

sudo /usr/bin/vim -c ':!/bin/sh'

Key Takeaways & Lessons Learned

  • [Skill/Technique]: Webpage enumeration, Reverse-Tabnabbing
  • [Vulnerability Type]: We were able to take advantage of scripts that automates mundane tasks to elevate our privilege level-by-level.
  • [Tool/Methodology]: Using my remote web server and netcat port listeners allowed me to get an initial foothold. Re-using netcat allowed me to laterally move to another user with more privileges.
